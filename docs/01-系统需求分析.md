# 应急管理知识库向量数据库系统 - 系统需求分析
## 1. 项目概述
本项目是一款基于向量数据库的应急管理智能检索系统，核心目标是解决应急知识检索的高效性、准确性问题，通过自然语言语义检索，快速匹配最相关的应急处置方案，为用户提供便捷、专业的应急知识查询服务。系统采用现代化前端界面+后端API+向量数据库架构，支持应急知识的存储、语义检索，适配个人及小型组织的应急知识管理需求，可直接用于课程设计、小型项目演示。

## 2. 功能需求
### 2.1 核心功能需求
- 应急知识向量存储：支持将应急场景（火灾、地震、溺水、触电等）的知识文本，通过嵌入模型转换为向量，存入向量数据库，实现知识的结构化存储。
- 自然语言语义检索：用户通过前端界面输入自然语言问题（如“有人触电怎么办”），系统调用向量数据库进行语义匹配，返回最相关的应急方案，并显示匹配度分数。
- 可视化交互界面：提供高颜值、易操作的前端界面，支持问题输入、检索触发、结果展示，包含深色/浅色视觉适配、动画交互效果，提升用户体验。
- API接口服务：封装向量数据库的核心操作（检索）为API接口，供前端界面调用，支持后续功能扩展（如批量添加知识、知识删除等）。

### 2.2 辅助功能需求
- 结果排序：检索结果按语义匹配度从高到低排序，清晰展示匹配分数，帮助用户快速定位最优答案。
- 异常处理：针对空查询、无匹配结果等场景，提供友好的提示信息，提升系统健壮性。
- 跨浏览器兼容：支持主流浏览器（Chrome、Edge、Firefox），保证界面正常显示和功能正常运行。

## 3. 非功能需求
### 3.1 性能需求
- 检索响应速度：用户输入问题后，系统响应时间≤1秒，检索结果快速展示，无明显延迟。
- 稳定性：系统连续运行无崩溃，支持多轮检索操作，向量数据库存储、读取稳定。
- 并发需求：支持单用户多轮检索，满足个人及小型团队的使用场景，无并发卡顿。

### 3.2 易用性需求
- 界面简洁直观：前端界面布局清晰，操作流程简单，用户无需学习即可完成问题输入和检索操作。
- 交互友好：包含按钮 hover 动画、输入提示、结果卡片过渡效果，提升用户操作体验。
- 可维护性：代码模块化设计（前端界面、API服务、向量数据库操作分离），便于后续功能扩展和bug修复。

### 3.3 技术需求
- 开发语言：Python（后端）、HTML/CSS/JavaScript（前端）。
- 核心框架：FastAPI（后端API）、Chroma（向量数据库）、Sentence-Transformers（文本嵌入模型）。
- 运行环境：Windows系统，Python 3.8及以上版本，支持浏览器访问（Chrome 90+优先）。

## 4. 用户需求
本系统的目标用户为需要查询应急知识的个人、学生及小型组织，用户需求主要包括：
1.  快速获取准确的应急处置方案，无需手动翻阅大量文档。
2.  操作简单，无需掌握专业技术，通过自然语言即可完成检索。
3.  界面美观，交互流畅，提升使用体验。
4.  系统稳定，检索结果可靠，满足应急场景下的快速查询需求。

## 5. 需求确认
本系统所有需求均围绕“应急知识语义检索”核心，贴合向量数据库的应用场景，兼顾功能完整性和易用性，符合课程设计、小型项目的需求标准，可直接用于演示和交付。
